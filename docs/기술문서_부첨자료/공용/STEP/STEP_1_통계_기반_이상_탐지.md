### 통계 기반 이상 탐지 (Statistical Anomaly Detection)란?
평소 값이 움직이는 범위를 계산해두고, 그 범위를 벗어나면 이상이라고 판단하는 방법입니다

어떤 복잡한 인공지능 모델이 아니라,
✔ 평균을 계산하고
✔ 얼마나 흔들리는지 계산하고
✔ 많이 벗어나면 이상이라고 정하는 방식입니다.

더 쉽게 예를 들어 보겠습니다.
- 평소 센서 값이 100 근처에서 움직입니다.
- 보통 98 ~ 102 사이에서 왔다 갔다 합니다.

그런데 갑자기 120이 나왔다면?
- 이건 평소 범위를 많이 벗어났네?”
- 이상일 가능성이 높다.
이렇게 판단하는 방식이 통계 기반 이상 탐지입니다.


🔹 핵심 원리
통계 기반 이상 탐지는 이렇게 동작합니다.

1️⃣ 정상 상태를 숫자로 표현합니다. (평균, 표준편차 등)
2️⃣ 현재 값이 그 정상 상태에서 얼마나 벗어났는지 계산합니다.
3️⃣ 미리 정해둔 기준을 넘으면 이상으로 판단합니다.


🔹 어떤 기준을 사용하나요?
주로 이런 것들을 사용합니다.
- 평균 (평소 중심 값)
- 표준편차 (얼마나 흔들리는지)
- 분산
- 분위수 (상위 몇 % 기준)
- 확률 분포

어렵게 들리지만 핵심은 이것입니다
평소 패턴을 숫자로 계산하고, 벗어나면 이상치

🔹 이상치 판단의 기본 생각
통계 기반 탐지의 철학은 아주 단순합니다.
✔ 정상 데이터는 일정한 범위 안에 있다
✔ 그 범위를 벗어나면 비정상이다

즉, 정상 범위를 계산하고 그 밖으로 나가면 이상이라고 보는 방식입니다.

🔹 자주 쓰이는 방법들
실무에서 많이 사용하는 방식은 다음과 같습니다.
- Z-score 방식
- 이동 평균 / 이동 표준편차 방식
- IQR (사분위 범위)
- 단순 임계값 규칙
- 확률 기반 판단
- 기준선 계산 → 벗어났는지 확인

🔹 실시간 관제 시스템에서는 어떻게 쓰이나요?
예를 들어 공장 설비를 모니터링한다고 생각해보겠습니다.

센서 값이 계속 들어옵니다.
시스템은 이렇게 동작합니다:
```
데이터 들어옴
→ 최근 데이터로 평균 계산
→ 현재 값이 얼마나 벗어났는지 계산
→ 기준 넘으면 알람 발생
```
즉, 것은 모델이 아니라 판단 로직(규칙 엔진)입니다.

###### 🔹 머신러닝 기반 이상 탐지와의 차이
| 구분       | 통계 기반 | 머신러닝 기반        |
| -------- | ----- | -------------- |
| 학습 필요    | 없음    | 필요             |
| fit() 사용 | 안 함   | 함              |
| 해석       | 매우 쉬움 | 비교적 어려움        |
| 계산 속도    | 매우 빠름 | 상황에 따라 느릴 수 있음 |
| 데이터 양    | 적어도 됨 | 많이 필요          |
통계 기반은: 공식 + 규칙 + 계산

머신러닝은: 데이터를 학습해서 패턴을 배움입니다.

통계기반을 실무에서 많이 쓰는 이유는:
✔ 빠르고
✔ 가볍고
✔ 이해하기 쉽고
✔ 설명하기 쉽기 때문입니다.

그래서 다음과 같은 곳에서 많이 사용됩니다.
- 센서 이상 감지
- 서버 장애 탐지
- 트래픽 급증 감지
- 금융 거래 이상 감지
- 제조 설비 모니터링

이상 판단을 하는 처리 순서(로직 흐름)
1️⃣ Rolling Mean (이동 평균)
최근 N개 평균을 계산합니다. → 지금의 정상 기준선입니다.

2️⃣ Rolling Std (이동 표준편차)
최근 N개가 얼마나 흔들리는지 계산합니다. → 변동성 기준입니다.

3️⃣ Z-score 계산
현재 값이 평균에서 몇 배 벗어났는지 계산합니다.

예:
- 0에 가까우면 정상
- ±3 이상이면 이상 가능

4️⃣ 임계값 판단
미리 정해둔 기준(예: 3)을 넘으면 이상으로 판단합니다.

5️⃣ 알람 발생
이상이라고 판단되면:
✔ ALERT 상태 전환
✔ 이벤트 기록
✔ 관리자 알림 이렇게 이어집니다.

통계 기반 이상 탐지는 평소 데이터의 범위를 계산해두고,
그 범위를 크게 벗어나면 이상으로 판단하는 규칙 기반 탐지 방식입니다.

###### 이상 판단 방식과 시계열 예측 모델의 차이
| 구분  | 통계 기반 이상 탐지 | Naive / ARIMA  |
| --- | ----------- | -------------- |
| 목적  | 이상 여부 판단    | 미래 예측          |
| 학습  | 없음          | 있음 (ARIMA는 특히) |
| 출력  | 정상 / 이상     | 다음 값 예측        |
| 역할  | 경보 엔진       | 예측 모델          |
###### 정상성 검사와 통계 기반 이상 탐지의 개념 비교
| 구분  | 통계 기반 이상 탐지   | 정상성 검사        |
| --- | ------------- | ------------- |
| 목적  | 지금 값이 비정상인가?  | 데이터가 안정적인가?   |
| 방식  | 규칙 기반 판단      | 통계 검정 알고리즘    |
| 예   | Z-score, IQR  | ADF, KPSS     |
| 질문  | 지금 이 값은 이상인가? | 이 시계열은 정상적인가? |

### 패턴 뼈대코드
최근 기준선 계산 → 얼마나 튀었는지 수치화 → 기준 초과 시 알람 발생 과정입니다.
```python
# ============================================
# [STEP 1 Skeleton] 통계 기반 이상 탐지 패턴
# Rolling Mean / Rolling Std / Z-score / Alert
# ============================================

import numpy as np
import pandas as pd

# (0) 데이터 준비
# df: columns = ["timestamp", "value"]
df = df.sort_values("timestamp").reset_index(drop=True)

# (1) Rolling window 설정 (예: 최근 30개)
WINDOW = 30

# (2) Rolling 통계 계산
df["roll_mean"] = df["value"].rolling(window=WINDOW, min_periods=WINDOW).mean()
df["roll_std"]  = df["value"].rolling(window=WINDOW, min_periods=WINDOW).std(ddof=0)

# (3) Z-score 계산 (표준편차가 0인 경우 방지)
EPS = 1e-9
df["z_score"] = (df["value"] - df["roll_mean"]) / (df["roll_std"] + EPS)

# (4) 임계치(Alert) 규칙
Z_THRESHOLD = 3.0
df["is_anomaly"] = df["z_score"].abs() >= Z_THRESHOLD

# (5) 결과 확인
# - 최근 구간에서 알람이 언제 울렸는지 확인
anomalies = df[df["is_anomaly"]]
print(anomalies[["timestamp", "value", "roll_mean", "roll_std", "z_score"]].tail(10))
```

해석:
`sort_values("timestamp")` : 시간순으로 정렬
- `df` 안에는 `timestamp`(시간)과 `value`(센서값)가 있으면
- 이상탐지(rolling mean/std, z-score)는 “이전 값들(최근 30개)”을 기준으로 계산합니다.
- 그런데 데이터가 시간순이 아니라 섞여 있으면?

###### 예를 들어 데이터가 이렇게 섞여 있으면:
|timestamp|value|
|---|---|
|10:00:03|101|
|10:00:01|99|
|10:00:02|100|
무조건 시간 기준 오름차순 정렬을 해줍니다.

`reset_index(drop=True)` : 인덱스를 0부터 다시 깔끔하게 재설정
만약 오름차순 정렬한 인덱스가 그대로 남아있다면 인덱스가 꼬일수 있으므로 다시 정렬합니다.
예시: 정렬전 0,1,2,3... 정렬후 2,3,1...  이항태가 되면 디버깅시도 헷갈리고 혼란이 있으므로 정렬된 기준으로 다시 인덱스를 재설정 합니다.

즉, 이단계는 시계열 데이터가 ‘진짜 시간 순서’대로 처리되도록 정렬하고, 이후 작업이 헷갈리지 않도록 인덱스를 깔끔하게 다시 매기는 초기 준비 단계입니다.

Rolling window 설정이란?
최근 데이터 몇 개를 기준으로 계산할지 정하는 것으로 센서 데이터는 계속 변합니다.
```
과거 전체 평균 → 의미 없음
지금 상태 기준 → 중요
```
그래서 우리는 이런 질문을 합니다. 최근 값들 기준으로 보면 정상인가?
이때 사용하는 것이 Rolling Window입니다.

Rolling Window를 그림으로 이해
WINDOW = 3 이라고 가정

데이터
```
[10, 11, 9, 10, 50]
```

Rolling 계산 방식:
```
(10,11,9) → 평균 계산
(11,9,10) → 평균 계산
(9,10,50) → 평균 계산
```
✔ 창(window)이 옆으로 이동하면서 계산
✔ 항상 “최근 N개”만 봄

(1) Rolling window 설정
예시: 센서가 1초마다 들어오면:
```
WINDOW = 30 → 최근 30초 의미
```

왜 전체 평균 안 쓰고 Rolling 쓰나?

전체 평균 문제:
❌ 과거 오래된 값 영향 큼
❌ 상태 변화 반영 느림
❌ 실시간 시스템에 부적합

Rolling 평균:
✅ 최근 상태만 반영
✅ 시스템 현재 상태 추적 가능
✅ 이상 탐지에 적합

관제 시스템 관점에서 해석
Rolling Window는 사실상 “현재 정상 상태 기준선을 만드는 장치” 입니다.
즉, 최근 데이터 묶음 → 정상 패턴 추정 하여 그 기준으로 이상 판단을 합니다.

---
(2) Rolling 통계 계산
```python
df["roll_mean"] = df["value"].rolling(window=WINDOW, min_periods=WINDOW).mean()
df["roll_std"]  = df["value"].rolling(window=WINDOW, min_periods=WINDOW).std(ddof=0)
```
	최근 WINDOW 구간의 정상 기준을 만드는 코드

이 코드가 하는 일은 각 시점마다 최근 WINDOW개 값의 평균(roll_mean)과 표준편차(roll_std)를 계산해서, 현재 값이 정상 범위인지 판단할 기준을 만듦니다.

`df["value"].rolling(...)` : `rolling`은 “움직이는 창(window)” 을 뜻해요
- `window=WINDOW` → 창의 크기(예: 30개)
- 창이 한 칸씩 앞으로 이동하면서,
- 그 창 안에 있는 값들로 통계를 계산합니다.

위에서 설명한 대로 예를 들어 WINDOW=3이라면:
value가 `[10, 11, 9, 10, 50]`일 때
- 첫 창: `[10, 11, 9]`
- 다음 창: `[11, 9, 10]`
- 다음 창: `[9, 10, 50]`

이렇게 “최근 3개씩” 묶어서 계산해요.

`mean()` : Rolling Mean (최근 평균)
```python
...rolling(...).mean()
```
	지금 시점 기준으로 최근 WINDOW개 값의 평균

`std(ddof=0)` : Rolling Std (최근 표준편차)
```python
...rolling(...).std(ddof=0)
```

표준편차란 흔들림 정도(변동성)입니다.

각 시점에서의 의미: 최근 WINDOW개 값이 평균 주변에서 얼마나 흔들리는지
- 흔들림이 작으면 std가 작음 → 작은 변화도 이상으로 볼 수 있음
- 흔들림이 크면 std가 큼 → 변화에 더 관대해짐
그래서 Z-score가 가능합니다. 평소에 많이 흔들리는 센서와 거의 안 흔들리는 센서를 공정하게 판단합니다.

---
(3) Z-score 계산 : (표준편차가 0인 경우 방지)
```python
EPS = 1e-9
df["z_score"] = (df["value"] - df["roll_mean"]) / (df["roll_std"] + EPS)
```

`1e-9` 가 뜻하는 값
- 과학적 표기법 (Scientific Notation)로 `e` = exponent (지수)와 `-9` = 10의 -9승으로 표기하며
- `1e-9` =0.000000001 (10의 -9승)
- 아주아주 작은 숫자(거의 0에 가까움)

위의 계산은 표준편차(roll_std)가 0이 되는 순간, 나눗셈이 터지는 걸 막는 안전장치입니다
- 분모가 0이면 `(어떤 값) / 0` 이 되어 계산이 불가능(에러/무한대)이 됨
- 그래서 `roll_std`에 아주 작은 수(EPS)를 더해서 분모가 0이 되는 상황을 피함
- EPS는 수의 범위가 아니라
- 0으로 나누는 오류 방지용 상수(안전핀)입니다.
- 목적은 거의 0인데 계산에는 문제 안 생기게 하는 값입니다.

---
(4) 임계치(Alert) 규칙
```python
Z_THRESHOLD = 3.0
df["is_anomaly"] = df["z_score"].abs() >= Z_THRESHOLD
```
정상 법위의 기준을 3으로 두고

`df["z_score"]` : 현재 값이 평균에서 얼마나 튀었는지의 점수
`.abs()` : 절대값 함수로 (+4)위로 튀어도 이상, (-4)아래로 튀어도 이상이므로 부호제거
`>= Z_THRESHOLD` : 이 정도 이상이면 경보 울려라
- threshold = 3
- z_score = 2.5 → 정상
- z_score = 3.2 → 이상

---
(5) 결과 확인 - 최근 구간에서 알람이 언제 울렸는지 확인
```python
anomalies = df[df["is_anomaly"]]
print(anomalies[["timestamp", "value", "roll_mean", "roll_std", "z_score"]].tail(10))
```
이 구간에서 이상수치 구간만 필터하여 끝에서 10개 데이터만 리스트에 있는 데이터 기준으로 출력합니다.

### 실무를 위한 연습 전체코드
```python
import numpy as np
import pandas as pd
import plotly.express as px

# -------------------------------------------------
# (0) 데이터 준비 (실습용 센서 데이터 생성)
#    - df: columns = ["timestamp", "value"]
# -------------------------------------------------
np.random.seed(42)

n = 600  # 10분치(예: 1초 간격 600개)
timestamps = pd.date_range("2026-02-11 10:00:00", periods=n, freq="s")

# 정상 구간: 평균 100, 노이즈 약간
values = 100 + np.random.normal(0, 1.2, size=n)

# 이상치(스파이크) 일부 삽입
spike_idx = [120, 121, 260, 400, 520]
values[spike_idx] += [12, -14, 18, -20, 15]

df = pd.DataFrame({"timestamp": timestamps, "value": values})

# (실시간 데이터처럼 쓰려면, 항상 시간순 정렬이 기본)
df = df.sort_values("timestamp").reset_index(drop=True)


# -------------------------------------------------
# (1) Rolling window 설정 (예: 최근 30개 = 최근 30초)
# -------------------------------------------------
WINDOW = 30          # 최근 30초(1초 간격이면 30개)
Z_THRESHOLD = 3.0    # 보통 3σ 규칙(상황에 따라 2.5~4 조절)

# -------------------------------------------------
# (2) Rolling 통계 계산 (rolling mean / rolling std)
#    - roll_mean: 최근 WINDOW 구간의 평균
#    - roll_std : 최근 WINDOW 구간의 표준편차(변동성)
# -------------------------------------------------
# min_periods=WINDOW:
# - WINDOW개 미만일 때는 통계를 계산하지 않음(NaN)
# - 초반 구간은 통계가 불안정하므로 알람을 안 울리게 하는 안전장치
df["roll_mean"] = df["value"].rolling(window=WINDOW, min_periods=WINDOW).mean()
df["roll_std"]  = df["value"].rolling(window=WINDOW, min_periods=WINDOW).std(ddof=0)

# -------------------------------------------------
# (3) Z-score 계산
#    - z_score = (현재값 - 최근평균) / 최근표준편차
#    - "현재 값이 평균에서 표준편차 몇 배나 떨어져 있는가?"를 수치로 표현
# -------------------------------------------------
# 표준편차가 0에 가까우면 분모가 0이 되어 계산이 터질 수 있으므로 작은 값(EPS) 더함
EPS = 1e-9
df["z_score"] = (df["value"] - df["roll_mean"]) / (df["roll_std"] + EPS)

# -------------------------------------------------
# (4) 임계치(Alert) 규칙
#    - abs(z_score) >= Z_THRESHOLD 이면 이상치(경보)로 표시
# -------------------------------------------------
df["is_anomaly"] = df["z_score"].abs() >= Z_THRESHOLD

# -------------------------------------------------
# (5) 결과 확인 (알람 요약 + 알람 발생 시점 출력)
# -------------------------------------------------
alarm_count = int(df["is_anomaly"].sum())
print(f"알람 개수: {alarm_count}개 (Z_THRESHOLD={Z_THRESHOLD}, WINDOW={WINDOW})")

# 알람이 발생한 행만 따로 확인 (언제/얼마나 튀었는지)
alarm_rows = df[df["is_anomaly"]].copy()
print(alarm_rows[["timestamp", "value", "roll_mean", "roll_std", "z_score"]].head(10))

# -------------------------------------------------
# (선택) 시각화 1) 값(value) + 롤링 평균(roll_mean)
#    - 정상 구간에서는 value가 roll_mean 주변에서 움직임
#    - 스파이크가 있으면 선이 갑자기 튀는 모습이 보임
# -------------------------------------------------
fig1 = px.line(df, x="timestamp", y=["value", "roll_mean"], title="Sensor value vs Rolling Mean")
fig1.show()

# -------------------------------------------------
# (선택) 시각화 2) Z-score 그래프
#    - 임계치(Z_THRESHOLD)를 넘는 구간이 이상 신호 후보
# -------------------------------------------------
fig2 = px.line(df, x="timestamp", y="z_score", title="Z-score over time")
fig2.show()

# -------------------------------------------------
# (선택) 시각화 3) 알람 포인트(이상치)만 점으로 표시
#    - 실제 경보가 울리는 시점만 콕 찍어서 확인
# -------------------------------------------------
alarm_plot = df[df["is_anomaly"] & df["roll_mean"].notna()].copy()
fig3 = px.scatter(alarm_plot, x="timestamp", y="value", title="Anomaly points (Threshold Alert)")
fig3.show()
```

### 이 코드를 통해 무엇을 증명하거나 얻는가”?

✅ 얻는 것 1: 이상 탐지의 최소 자동화
- “누가 눈으로 보는 것”이 아니라
- 알고리즘이 자동으로 이상 시점을 표시한다

✅ 얻는 것 2: 이상 여부를 ‘숫자(z_score)’로 설명 가능
- “그냥 이상해 보여요”가 아니라
- “평균에서 4.8σ(시그마) 튀었습니다”라고 말할 수 있음
    → 팀 발표/로그/리포트에 강력함

`σ` = 표준편차로 데이터가 평균 주변에서 얼마나 퍼져 있는가(흔들림의 크기)를 나타내는 대표적인 지표입니다.
- `1σ` → 평균에서 표준편차 1배 거리
- `2σ` → 평균에서 표준편차 2배 거리
- `3σ` → 흔히 쓰는 이상치 기준
- `4.8σ` → 엄청 크게 튄 값 (강한 이상 신호)

✅ 얻는 것 3: 튜닝 포인트가 명확해짐
- WINDOW(최근 몇 초를 정상으로 볼지)
- THRESHOLD(얼마나 튀면 알람인지)

즉, 기준을 조절해서 민감도를 조절할 수 있는 틀을 얻습니다.

정상처럼 보이는 가짜 센서 데이터를 만들기 위한 생성 규칙으로 실험을 위한 데이터 시뮬레이션 도구로 정규 분포에서 가장 많이 쓰이는 근사 모델입니다.

```python
# 정상 구간: 평균 100, 노이즈 약간
values = 100 + np.random.normal(0, 1.2, size=n)
```
	평균 100 주변에서 살짝 흔들리는 정상 센서 값 생성
	100 근처에서 살짝 흔들리는 숫자 n개 만들어줘를 명령하는 코드입니다.

위의 코드는 가짜 센서 값 생성기입니다.
위의 값을 돌리면 랜덤하게 평균중심 아래와 같은 숫자가 나옵니다.
```
+0.5
-1.3
+0.8
-0.2
+2.1
-0.9
```
✔ 양수 나옴
✔ 음수 나옴
✔ 대부분 0 근처

`np.random.normal(0, 1.2, size=n)` : 이 함수의 의미는 정규분포(가우시안 분포)를 따르는 랜덤 값 생성기

파라미터 의미:
- `0` → 평균(mean)
- `1.2` → 표준편차(std)
- `size=n` → n개 생성

현실에서는 케이스마다 값이 다르게 나옵니다.
- 온도(섭씨) → 음수 가능
- 전압 → 보통 음수 거의 없음
- CPU 사용률 → 절대 음수 불가
- 압력 → 음수 거의 없음
- 로그 트래픽 → 음수 불가
```
values = 기준값(정상 상태) + 랜덤 흔들림
```
	100 근처로 맞추는 이유는 그냥 읽기 편하게 하기 위함입니다.

현실 데이터를 생각하며 이해를 해봅니다.
센서든 시스템이든 대부분 이런 구조입니다.
- 어떤 안정적인 정상 범위가 존재
- 값은 그 주변에서 조금씩 흔들림
- 문제가 생기면 갑자기 튐

예:
- 온도 센서 정상 → 24 ~ 26℃
- 서버 응답시간 정상 → 110 ~ 130ms
- 전류 정상 → 8 ~ 9A

➡ 항상 “중심값(평균)”이 존재합니다.
100은 그냥 ‘정상 중심값’을 표현한 것입니다. 100이라는 숫자에는 특별한 의미가 없어요.
✔ 보기 쉽고
✔ 계산 해석이 직관적이고
✔ 그래프에서 변화가 잘 보임

온도를 예시로 설명하면
1️⃣ 온도 센서 정상 → 24 ~ 26℃ 평균은 약 25 이것을 100 기준으로 옮기면
```
99 ~ 101
```
이걸 수학적으로 보면:
- 평균 = 25
- 변동 폭 = ±1
변동폭은 온도센서 정상 범위를 기준으로 아래와 같기때문에 ±1 이렇게 나옵니다.
- 25 - 1 = 24
- 25 + 1 = 26

중심을 100으로 옮긴다는 의미는 중심(평균)만 이동시키는 연산은 그냥 아래와 같습니다.
```
새 값 = 원래 값 + 이동량
```
여기서 이동량은 100 - 25 = +75

실제로 계산해 보면 원래 값들에 +75 적용
- 24 + 75 = 99
- 25 + 75 = 100
- 26 + 75 = 101
즉 결과는 99~101이 나옵니다.

그래서 결론은 25에서 ±1 만큼 흔들림이 있으나 평균 100에서 ±1 만큼 흔들림이 있으나 여전히 여전히 ±1 만큼 흔들립니다. 즉 위의 코드는 100근처의 정상 센서를 만들어내는 가짜 데이터 제조기라고 생각하면 됩니다.

이것은 이렇게 활용될 예정입니다
정상처럼 보이는 값들을 먼저 만들고, 최근 정상 범위(평균 + 변동성)를 계산한 뒤,
그 범위를 크게 벗어나는 값들을 이상으로 판단하여 알람을 줍니다.

- 121번째 값
- 260번째 값
- ...
여기만 건드리겠다는 뜻입니다.

`values[spike_idx] +=`는 그 위치 값들을 갑자기 크게 변경하여 어떤 값에 +12 → 확 증가시키고 어떤 값에 -14 → 확 감소 시켜서 이상 수치를 만들어 냅니다.

3이란 수치는 이상범위를 가상으로 만들어준 것이며 실제 센서데이터마다 이상수치의 범위가 지정되어 있어 그 수치를 시그마에 대입하면 됩니다.

---
최근 30초의 정상 기준선을 계산합니다.
- `roll_mean` = 최근 30초 평균 → 지금 정상이라면 이 정도
- `roll_std` = 최근 30초 표준편차 → 정상 범위에서 흔들리는 폭

그리고 `min_periods=WINDOW`를 둬서 초반 데이터(30개 전까지)는 계산하지 않게 함.

의미: 아직 데이터가 충분하지 않은 초반엔 기준이 불안정하니까 경보를 울리지 않게합니다

(2) Rolling 통계 계산 (rolling mean / rolling std)
```python
df["roll_mean"] = df["value"].rolling(window=WINDOW, min_periods=WINDOW).mean()
df["roll_std"]  = df["value"].rolling(window=WINDOW, min_periods=WINDOW).std(ddof=0)
```
	최근 WINDOW개 값으로 평균과 흔들림 크기를 계속 계산한다

- `window=WINDOW` : 최근 몇 개 데이터를 기준으로 계산할 것인가(계산범위설정)
- `min_periods=WINDOW` : 데이터가 30개 쌓이기 전까지는 계산하지 마라
- `.mean()` : 평균 계산 함수

---
지금 값이 “최근 기준”에서 얼마나 이상한지 점수로 만듦니다 (Z-score)
- `z_score = (현재값 - 최근평균) / 최근표준편차`
- 즉, 평균에서 표준편차 몇 배나 벗어났나?

EPS는 `roll_std`가 0에 가까울 때 분모가 0이 되는 걸 막는 안전장치.

의미: 그냥 튄 것 같다는 느낌이 아니라, ‘정량 점수’로 지금이 얼마나 튀었는지 계산합니다


(3) Z-score 계산
```python
EPS = 1e-9
df["z_score"] = (df["value"] - df["roll_mean"]) / (df["roll_std"] + EPS)
```
	표준편차(roll_std)가 0이 되는 순간, 나눗셈이 터지는 걸 막는 안전장치

뼈대 패턴에서 설명한 내용

---
규칙을 적용해서 이상이다/아니다를 결정합니다.
- `abs(z_score) >= 3` 이면 `is_anomaly=True`

의미: 점수가 기준선을 넘으면, 이 시점은 경보를 울릴 이벤트로 확정.


(4) 임계치(Alert) 규칙
```python
df["is_anomaly"] = df["z_score"].abs() >= Z_THRESHOLD
```
	Z-score가 기준값보다 크면 이상(True)으로 표시

`df["z_score"]` : 현재 값이 평균에서 얼마나 튀었는지의 점수
`.abs()` : 절대값 함수로 (+4)위로 튀어도 이상, (-4)아래로 튀어도 이상이므로 부호제거
`>= Z_THRESHOLD` : 이 정도 이상이면 경보 울려라
- threshold = 3
- z_score = 2.5 → 정상
- z_score = 3.2 → 이상

해석
```
roll_mean → NaN
roll_std → NaN
z_score → NaN
```
	아직 30개(WINDOW) 데이터가 안 쌓였기 때문

아래쪽 숫자 부분 해석
```
value = 100.45

```
의미:
✔ 현재 값이 평균 근처
✔ 거의 안 튐
✔ 정상 범위

Z-score는 3까지 정상 범위로
- z_score = 0.3 → 정상
- z_score = 1.2 → 정상
- z_score = 2.8 → 정상
- z_score = 3.1 → 이상
이상 판정 조건:
```
abs(z_score) >= 3
```
절대값(abs)을 사용하므로 정상 범위 구간은
```
-3 < z_score < +3
```
이 구간은 정상입니다.

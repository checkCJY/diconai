# 알람 시스템 재설계 — 최종 구현 계획 (D-30 시연 전)

## Context

PR #57 (d3dec78) 까지의 알람 작업이 원하는 결과가 아니었다는 피드백. 단순 UI 수정이 아니라 **알람 라우팅 정책 + UX 일관성** 차원의 재설계가 필요.

### 사용자가 새로 제기한 요구
1. 위험 지속 중 "확인 완료" 후에도 위험이 계속되면 팝업이 다시 떠야 함
2. 대시보드 외 다른 페이지에서도 위험 알람 팝업이 즉시 떠야 함
3. 알람 발생 비즈니스 로직에 권한 분기 — 관리자=전체 / 작업자=본인 지오펜스 영역만 + 대피 안내

### 팀 피드백 반영 (회색지대 5개 + 검증 4개)
- ACKNOWLEDGED 의미 = user-scoped ("본 사람만 안 보임, 다른 사용자에게는 계속 뜸")
- 5초 group 윈도우는 위험 한정 **1초 단축** (완전 해제 X — 노이즈 보호)
- 재팝업 쿨다운 환경변수화 `ALARM_REPOPUP_COOLDOWN_SEC` (운영 60s, 시연 15s)
- RESOLVED 시 떠있는 팝업 **자동 닫힘**
- target_user_ids = **동적 계산** (broadcast 시점, 사후 진입 작업자도 받음)
- current_geofence=None 작업자 정책 = **공장 전체 DANGER 만**
- admin-panel UX = **우상단 토스트 + 사이드 패널 + DANGER 10초 격상 모달**
- Phase 2 시 작업자 화면 **stub (공장 전체 위험 임시 push)** — Phase 3 전까지 시연 비공백
- 검증: 작업자 시연 두 번째 단말 / WS catch-up / 시연 시나리오 스크립트 / 머지 충돌 — 아래 별도 섹션

### 시연 일정
D-30 (2026-06-14). PG 마이그레이션은 알람 파일을 만지지 않음 (확인 완료). 머지 충돌 위험 없음.

### 추가로 확인된 사실 (팀 피드백 후 조사)
- **Event 모델은 user-scoped ACKNOWLEDGED 미지원** — `acknowledged_by` 가 단일 FK. → `EventAcknowledgement(event, user, acknowledged_at, unique_together)` 신규 모델 + 마이그레이션 필요
- **WS 재연결 catch-up 미구현** — 끊김 중 알람 영구 손실. `since=timestamp` 파라미터 + AlarmRecord 24h 쿼리 추가 필요 (~50줄)
- **머지 충돌 위험 없음** — 알람 관련 로컬 브랜치는 모두 stale (PR #56·#57 머지 후 갈라짐). PG 마이그레이션도 알람 파일 미터치

## 시연 시나리오 스크립트 (D-30)

결정의 빈틈 드러내는 도구. 시나리오 진행하면서 결정이 어색하면 보완.

### S1 — 관리자 화면: 가스 위험 발생 → 확인 → 위험 지속 → 재팝업
1. 관리자 대시보드 (`/dashboard/`) 접속 → 정상 상태
2. 가스 시뮬레이터로 CO 위험 발생 → **차단형 모달 팝업** (15초)
3. 관리자 "확인 완료" → 팝업 닫힘 + 백엔드 `EventAcknowledgement` 생성
4. 위험 지속 → 15초 (시연 모드) 후 새 AlarmRecord → **새 팝업 재표시**
5. 위험 해소 → 떠있는 팝업 자동 닫힘 + "위험 해소" 토스트 + 패널 정상화 그룹화

### S2 — 작업자 단말: 본인 지오펜스 위험 (Phase 3 시연 시)
1. 두 번째 단말 (또는 같은 PC 시크릿 창) 에 WORKER 계정 로그인
2. 작업자 페이지 → WS 연결 + 본인 위치(지오펜스 A) 확인
3. 지오펜스 A 안 가스 센서 위험 → 작업자 단말 **즉시 팝업** + "대피하세요"
4. 관리자 단말도 동시 수신 (전체 broadcast)
5. 지오펜스 B 의 알람은 작업자 단말에 안 옴 — 관리자만 수신

### S2' — Phase 2 (작업자 stub) 시연 시
- "작업자 단말에서 위험 알람이 옵니다. 단 본인 영역 한정 필터는 다음 스프린트입니다" 멘트로 stub 동작 설명

### S3 — admin-panel 사용자: 작업 중 알람
1. admin-panel 의 폼 작성 페이지 (지도 편집 등) 진입
2. 가스 위험 발생 → **우상단 토스트** (입력 손실 없음)
3. 10초 무응답 시 → **차단형 모달로 격상**
4. 사이드 패널에 알람 스택 누적 표시

### S4 — 다른 페이지에서 알람
1. event_detail / monitoring_realtime / admin-panel 각 페이지 → 알람 발생 시 즉시 팝업
2. 페이지 이동 중에도 WS 재연결 (`since=` catch-up) 으로 미수신 알람 보충

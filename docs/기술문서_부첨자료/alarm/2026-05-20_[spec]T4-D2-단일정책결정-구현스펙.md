## §3 선행 작업이 제공하는 자산 (D2 입력)

### 3.1 D1a — AI inference state machine

**모듈**: [fastapi-server/services/ai_mute.py](../../fastapi-server/services/ai_mute.py)

```python
from services.ai_mute import AIInferenceState, mark_ai_state, get_ai_state

# 5 state — process_anomaly_inference 안 마킹할 값
class AIInferenceState(str, Enum):
    FIRED           = "fired"
    INFERRED_NORMAL = "inferred_normal"
    INFERRED_FAILED = "inferred_failed"
    WARMING_UP      = "warming_up"
    DISABLED        = "disabled"

async def mark_ai_state(device_id, channel, data_type, state, ttl_sec=60) -> None
async def get_ai_state(device_id, channel, data_type) -> AIInferenceState | None
```

**마킹 위치 5 지점** (process_anomaly_inference 안):

| 위치 | 마킹할 state | 추가 호출 |
|---|---|---|
| 진입 시 `(channel, data_type) ∉ _INFERENCE_ENABLED_CHANNELS` | `DISABLED` | — |
| 윈도우 빌드 중 `len(win) < _INFERENCE_WINDOW` | `WARMING_UP` | — |
| IF 추론 성공 + `combined ∈ FIRE_LEVELS` (발화) | `FIRED` | + 기존 `mark_ai_recent()` 도 호출 (DRF AI mute 가드 호환 유지) |
| IF 추론 성공 + `combined == "normal"` | `INFERRED_NORMAL` | — |
| IF 추론 예외 | `INFERRED_FAILED` | — |

→ FIRED 만 mark_ai_recent 도 같이 호출 (`ai_fired:*` 키 set). 나머지 4 state 는 `ai_state:*` 만.

## §4 decide_alarm 매트릭스 (plan §5 발췌)

### 4.1 의사 코드

```python
from services.ai_mute import AIInferenceState, get_ai_state
from power.services.threshold_sync import get_threshold_meta
from core.constants import ALARM_SOURCE_REASON

_DRF_ITEM_BY_DATA_TYPE = {"watt": "power_w", "current": "current", "voltage": "voltage"}


def _evaluate_static(value, data_type, device_id, channel) -> str:
    """정격 % 환산 후 risk_level 반환. 캐시 미존재 시 'normal' (fail-safe)."""
    meta = get_threshold_meta(_DRF_ITEM_BY_DATA_TYPE[data_type])
    if not meta:
        return "normal"
    # ... rated_w/a/v 기반 % 환산 + warning_min/max·danger_min/max 평가
    # 단방향 (watt/current) 은 max 만, 양방향 (voltage) 은 min/max 양쪽


async def decide_alarm(device_id, channel, value, data_type, ai_result):
    """source 5종 매트릭스 분기. 발화 없음이면 None.

    Args:
        ai_result: AI 추론 결과 (combined_risk, anomaly_score, algorithm_source 등)
                   AIInferenceState 와 1:1 매핑.

    Returns:
        Alarm(alarm_type, source, reason, risk_level, ...) 또는 None
    """
    ai_state = await get_ai_state(device_id, channel, data_type)
    static_risk = _evaluate_static(value, data_type, device_id, channel)
    static_fired = static_risk in ("warning", "danger")

    # AI 가 발화 → AI 알람 (정적 silent)
    if ai_state == AIInferenceState.FIRED:
        return _build("power_anomaly_ai", "ai", ...)

    # AI 비활성 채널 + 정적 발화 → 일반 룰 알람
    if ai_state == AIInferenceState.DISABLED and static_fired:
        return _build("power_overload", "static_no_ai_available", ...)

    # AI 워밍업 + 정적 발화 → safety net
    if ai_state == AIInferenceState.WARMING_UP and static_fired:
        return _build("power_overload", "static_cover_warmup", ...)

    # AI 추론 실패 + 정적 발화 → 정적 폴백
    if ai_state == AIInferenceState.INFERRED_FAILED and static_fired:
        return _build("power_overload", "static_cover_inference_fail", ...)

    # AI 정상 + 정적 발화 → AI 미탐 의심 (false negative 후보)
    if ai_state == AIInferenceState.INFERRED_NORMAL and static_fired:
        return _build("power_overload", "static_cover_miss", ...)

    # get_ai_state None (Redis 장애·만료) — DISABLED 동등 fail-safe 분기 권장
    if ai_state is None and static_fired:
        return _build("power_overload", "static_no_ai_available", ...)

    return None  # 발화 조건 미달
```

### 4.2 운영자 화면 매트릭스 (decide_alarm 결과)

| AI 상태 | 정적 결과 | source | 운영자 인지 |
|---|---|---|---|
| FIRED | * | `ai` | "AI 가 잡았다" |
| INFERRED_NORMAL | fired | `static_cover_miss` | "AI 가 놓쳤을 수도" |
| INFERRED_NORMAL | not fired | (no alarm) | (정상 — 가장 흔한 케이스) |
| INFERRED_FAILED | fired | `static_cover_inference_fail` | "AI 가 못 돌았다" |
| DISABLED | fired | `static_no_ai_available` | "AI 영역 외 채널" |
| WARMING_UP | fired | `static_cover_warmup` | "AI 워밍업 동안 정적룰이 잡음" |
| * | not fired | (no alarm) | (정상) |

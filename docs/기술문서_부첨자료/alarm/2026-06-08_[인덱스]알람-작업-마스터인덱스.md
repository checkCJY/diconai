## 5. 검증 메모 (2026-06-08 실측)

> AI 직접 push 경로 latency를 처음 실측. "AI가 Celery보다 빠르다"는 **미검증·메커니즘 오류**로 확인.

| 구간 | p50 | p95 | p99 | 비고 |
|---|---|---|---|---|
| AI 직접 push (ingress→push) | 100ms | 198ms | 428ms | 추론 7.6ms + XADD 0.5ms + **DRF forward ~92ms** |
| 룰 Celery 큐대기 | 14ms | 46ms | 49ms | AI가 회피하는 부분 |
| 룰 Celery 태스크(DB+WS push HTTP) | 11ms | 700ms | 940ms | 꼬리 = WS-push HTTP 왕복 |

**결론**: AI 경로는 push 전에 `event_id` 확보를 위해 DRF forward를 **0.5s race로 동기 대기**(실측 ~92ms 지배) → "DRF 건너뛰고 즉시 push"가 아님. 진짜 강점 = **단일 결정자**(race 제거) + 정적 cover(fail-safe) + in-process push(룰의 WS-push HTTP 꼬리 700ms 회피). **"빠르다"는 기술문서에서 빼야 함.** (한계: 중간 부하·큐 미포화, e2e는 브라우저 미접속으로 미측정)

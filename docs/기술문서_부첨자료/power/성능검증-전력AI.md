## 1. 정확도 (라벨 vs AI 오프라인 재현)

출처: [../perf/2026-06-06_[검증]AI이상탐지-방법과결과.md](../perf/2026-06-06_[검증]AI이상탐지-방법과결과.md). 전력 채널당 86k~106k행. **방법 = 시뮬레이터 라벨(`is_anomaly`) vs 동일 모델·피처 오프라인 재현**.

| 축/시나리오 | recall | precision | FP율 | 비고 |
|---|---|---|---|---|
| 전력 — IF 단독 | 0.22~0.43 | 0.55~0.68 | 0.4~0.7% | |
| 전력 — 5축 (야간 포함) | 0.869 | 0.052 | 33.9% | |
| **전력 — 5축 (야간 제외)** ★ | **0.854** | 0.314 | **3.2%** | **운영 대표값** |
| degradation / overload / voltage_drop | 0.85~1.00 | | | overload는 threshold 룰 주력 |
| phase_loss / motor_stuck | 0.24~0.26 / 0.06~0.19 | | | **사각지대** |

**★ 야간격상(night_abnormal) ablation**: 야간 축 = *이상 0건 잡고 정상 26,259건 발화* → **탐지 기여 0·오탐 31%**. 제거 시 **FP 33.9%→3.2%(≈10배↓), recall 0.869→0.854 유지**.

## 2. 리드타임 (조기경고 선행성)

출처: [2026-06-08_[검증]전력AI-리드타임.md](2026-06-08_[검증]전력AI-리드타임.md). `MLAnomalyResult` 858,119행, clean 램프 4,444건.

| 채널/시나리오 | 선행률 | median | 비고 |
|---|---|---|---|
| 전체 (clean) AI `predict_warn` | 19% | 0.4s | mean 2.3s·p90 3.5s |
| ch1 압연기(급변) | 16% | 0.3s | clean의 92% 비중(평균 끌어내림) |
| ch9 / ch14 / ch15 | 48% / 51% / 62% | 3.5s / 3.4s / 2.3s | |
| **ch1 제외 3채널(drift성)** ★ | **~50%** | **3.4s** | |

**왜 전체값이 작은가**: 라벨 89%가 예측불가형(overload 52%·night 26%·motor_stuck 10%·전압 6%), drift는 degradation 8%뿐 → **모델 실패 아닌 데이터-목표 불일치**.

## 3. 코드 ↔ 검증 괴리 (반드시 인지)

- **야간격상**: 검증은 "제거 권고"인데 **코드는 활성** (`night_escalation.py` `_NIGHT_ESCALATION`, anomaly_inference.py:346-365). 마지막 수정 05-21 = 검증(06-06)보다 앞섬, 이후 변경 0.
- **ARIMA**: 코드상 정상 결합(`combine_risk_3axis`에 `arima_violation` 정식 입력). "기여≈0"은 **drift 데이터 부재 탓**(코드 결함 아님) → 제거 판단 보류.
- **CP > Z**: 실제 코드는 Change Point를 Z-score보다 먼저 검사(`risk_combine.py:151-154`). docstring(Z>CP)은 stale.
- **재현 신뢰성 확인됨**: 측정 피처 4종(value/roll_mean/roll_std/diff)·IF 설정(contamination 0.01, window 30)·활성채널(ch1/9/14/15)이 현재 코드와 일치.

## 4. 인용 주의 (전력 AI — 과대서술 금지)

1. **야간격상을 강점으로 쓰지 말 것** — 탐지 0·FP 31% 순수비용. night recall 1.00 = "빼도 다른 축이 100% 잡음".
2. "탐지율 87%"를 AI 성과로 단정 금지 — 야간 포함값 + overload는 threshold 룰. **운영 대표 = 야간 제외 0.854 / FP 3.2%**.
3. 리드타임 19%·0.4s를 AI 성능으로 쓰면 오해 — **"drift에서 ~50% 선행·median 3.4s"**. "초"는 시간압축 더미값 → **"danger 전 N틱 선행"**으로 표현.
4. ARIMA "쓸모없다" 단정 금지 — **미측정·보류**.
5. 모든 수치 = in-sample·합성·held-out 미적용 → **"지표용"** 표시. degradation 100% 등은 라벨 join 3.5% 매칭이라 더 약함.
6. `eval_anomaly_accuracy` 커맨드 **미존재** → 재현절차 필요(일회성 스크립트).

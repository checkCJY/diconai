# 전력 임계치 산업안전기준 정리

> **상태**: 초안 (2026-05-11) — 기준 문서. 코드/DB 적용은 후속 작업.
> **작성 맥락**: [`evaluate_power_risk()`](../../drf-server/apps/facilities/services/threshold_service.py) 가 현재 전력 W 단일 항목만 평가. 전류 A·전압 V 의 임계치 근거 부재. AI 이상탐지 스프린트와 병행 진행될 규칙 기반 보호 계층의 기준점 정의.

## 1. 목적과 범위

### 1.1 적용 대상
- [`PowerData`](../../drf-server/apps/monitoring/models/power_data.py) 의 `data_type` 별 위험도 평가 기준
  - `power_w` — 채널별 순시 전력 (W)
  - `current` — 채널별 전류 (A)
  - `voltage` — 채널별 전압 (V)

### 1.2 적용 제외
- 누설전류 / 접지저항 — PowerData 스키마 외, 누설전류 센서 추가 시 별도 기준 필요
- 차단기 동작 자체 — 펌웨어/스위치기어 영역
- 역률, 고조파(THD) — 본 시스템 수집 범위 외

### 1.3 ML 이상탐지와의 역할 분담
본 문서가 정의하는 임계치는 **명시적 안전 마지노선**만 다룬다. 시간대별 부하 패턴, 채널 간 상관 변화 같은 미세 이상은 AI 이상탐지 스프린트가 담당한다. 임계치 단계를 촘촘하게 잡으면 AI의 탐지 여지가 사라지므로, **2단계(warning/danger)** 로 단순화한다.

전력 AI는 **IF + ARIMA 조합** (un-downgrade — [`ai-model-study-2026-05-17.md §2.4`](study/ai-model-study-2026-05-17.md) 결론). IF는 패턴 이상, ARIMA는 예측 일탈을 각자 독립 판단해 룰과 함께 3중 보호 계층 형성.

| 구분 | 본 문서 (규칙 기반) | sklearn IF (패턴 이상) | statsmodels ARIMA (예측 일탈) |
|---|---|---|---|
| 잡는 것 | 정격 초과, 안전기준 위반 (알려진 위험) | 분포 outlier (변동성·차분 이상) | forecast 신뢰구간 위반, trend break, seasonal 이탈 |
| 근거 | KEC, KOSHA Guide, IEC | 운전 데이터 분포 | 시계열 자기상관·예측 모델 |
| 단계 | 2단계 (warning, danger) | 연속값 (anomaly score) | 예측값 ± 신뢰구간 (95%) |
| 산재 PoC 강점 | 설명 가능 ("정격 90% 초과") | 사전 학습된 정상 패턴 ↔ 신규 이상 | "예측 50W±10 대비 실측 200W" — 운영자 해석 + 선행 경보 |
| 도입 시점 | 즉시 (현재) | 1단계 PoC (현재 ch1·watt) | 2단계 (시연 후, D+30~) |

> **가스 도메인은 다른 결정**: 가스는 사고 시간 척도가 분 단위 즉시 위험 → ARIMA의 forecast·seasonal·multi-step 가치가 도메인에서 불필요 → ARIMA를 IF 피처로 격하 채택. 도메인 의존 결정으로 전력과 별개 경로 ([`ai-model-study-2026-05-17.md §2.4`](study/ai-model-study-2026-05-17.md)).

## 2. 참고 산업안전기준

| 기준 | 영역 | 핵심 수치 | 본 문서 활용 |
|---|---|---|---|
| KEC 212 (과전류 보호) | 과부하/단락 | 도체 허용전류 = 부하전류 × 1.25 이상, 차단기 정격 = 도체 허용전류 ≤ | 전력 W, 전류 A 의 정격 % 기반 상한 |
| KEC 232.3.9 (전압강하) + IEC 60038 | 저압 회로 전압 | KEC: 사용 전 전압강하 ≤ 5% (인입~말단 누적, 저압 옥내). IEC 60038: 저압 시스템 허용 ±10%, 권장 운전 범위 ±5% | 전압 V 의 정격 % 기반 양방향 — warning ±5% (IEC 권장 운전 범위), danger ±10% (IEC 허용 한계) |
| KS C IEC 60947-2 | 저압 차단기 | 정격전류 In, 동작전류 1.13·In(t시간 미동작) / 1.45·In(t시간 내 동작) | warning 80% / danger 100% 의 경험적 근거 |
| KOSHA Guide E-92-2011 | 전기설비 화재예방 | 전선·접속부 정격 80% 이내 운전 권장 (가이드 코드/제목은 채택 시점 재확인 권장) | warning_max 80% 채택 근거 |

> 인용 수치는 일반적으로 통용되는 기준이며, 실제 시공·검사 시에는 해당 시점의 KEC 개정본과 KS 본문을 직접 확인할 것.
>
> 산업안전보건기준에 관한 규칙 제301~310 조항이 전기재해 예방을 다루나, 임계 수치 자체는 충전부 방호·접지·정전작업 등 절차 영역이므로 본 문서 범위 외.
>
> **KEC 232.3.9 의 5% 와 IEC 60038 의 ±5% 관계**: KEC 5% 는 인입구~말단의 *누적* 전압강하 한계이고, 본 문서가 임계치로 사용하는 ±5% 는 *측정 지점에서의 사용 전압 권장 범위*(IEC 60038) 다. 모니터링은 분전반/말단 어딘가에서 측정되므로 후자가 직접적 근거이며, KEC 는 회로 설계 단계의 보완 근거로 인용한다.

## 3. 임계치 산정 원칙

### 3.1 절대값이 아닌 "정격 대비 %" 사용
- 압연기 100kW 와 송풍기 10kW 를 동일한 W 절대값으로 비교 불가
- IEC 60947 의 1.13·In / 1.45·In 동작 곡선이 이미 **정격 In 비율 기준**
- 정격값은 채널(설비)별로 다르므로 정격을 어딘가에 저장해야 함
- **80%-100% 의 양방향 정합성**: KEC 212 의 "도체 허용전류 = 부하전류 × 1.25" 와 본 문서 warning 80% (1/1.25 = 0.8) 는 동일 사고의 양면 표현이다. 한쪽은 설계 시 도체를 1.25배로 잡으라는 것이고, 다른 한쪽은 운전 시 정격 80%를 넘기지 말라는 것이다.

### 3.2 정격값 저장 위치 (후속 결정 사항)
현재 [`CHANNEL_TO_DEVICE`](../../fastapi-server/power/services/power_service.py) 는 채널→설비명 매핑만 있고 정격값은 없다. 두 가지 옵션:

- **옵션 A**: `PowerDevice` 모델에 `rated_w`, `rated_a`, `rated_v` 필드 추가 → 채널별 정격을 장비 메타로 보관
- **옵션 B**: `ThresholdGroup` 을 채널별로 분리 (`power_ch01`, `power_ch02`, ...) → 정격 % 가 아닌 절대값으로 채널별 임계치 정의

옵션 A 는 정격 % 라는 의도가 명확하고 모델이 단순. 옵션 B 는 기존 `Threshold` 스키마 변경 없이 가능. **권장은 A**이지만 본 문서 범위 외.

### 3.3 단계 구분 — 2단계로 단순화
[`evaluate_power_risk()`](../../drf-server/apps/facilities/services/threshold_service.py) 와 [`RiskLevel`](../../drf-server/apps/core/constants.py) 이 NORMAL / WARNING / DANGER 3단계만 사용. 본 기준도 동일하게 warning / danger 2단계만 정의.

### 3.4 양방향 임계치 (전압 전용)
전압은 강하(저전압)도 위험. [`Threshold`](../../drf-server/apps/facilities/models/thresholds.py) 모델이 이미 `warning_min`, `warning_max`, `danger_min`, `danger_max` 4필드를 보유하므로 **스키마 변경 없이 양방향 수용 가능**. 평가 패턴은 [`evaluate_gas_risk()` 의 O2 분기](../../drf-server/apps/facilities/services/threshold_service.py#L117-L124) 와 동일하게 구현 가능.

### 3.5 시간 조건과 히스테리시스 (운영 파라미터, 권고 기본값)

임계치 단일 지점 비교만으로는 0.5초 순시 스파이크에 의한 false positive 와 임계치 경계에서의 채터링이 발생한다. 본 문서가 정의한 임계치는 다음 운영 파라미터와 **함께** 적용될 때 PoC 단계에서 의미를 가진다.

- **DANGER 발화 조건** — 임계치 위반이 **3~5초 연속 지속** 시. 100% 단발 스파이크에 의한 오탐 방지. 현재 [`power_alarm.py`](../../drf-server/apps/monitoring/services/power_alarm.py) 의 `WARNING_DURATION_SEC=10` 정책과 정합되도록 추후 통합.
- **WARNING 발화 조건** — 임계치 위반이 **30~60초 연속 지속** 시.
- **히스테리시스 ±2%** — `warning_max` ↔ NORMAL 회복 시 임계치보다 2%p 낮은 지점에서 회복으로 판정 (예: warning_max 80% → 회복 78%). `danger ↔ warning`, `warning_min` 측 회복도 동일하게 2%p 갭 적용. 임계치 경계에서의 채터링 방지가 목적.

위 수치는 **PoC 권고 기본값**이며 확정 정책이 아니다. 운영 데이터 확보 후 어드민에서 채널별·시간대별로 조정 가능하다.

## 4. 항목별 임계치 표

### 4.1 과부하 영역 (단방향 상한)

| measurement_item | 단위 | warning_max | danger_max | 근거 |
|---|---|---|---|---|
| `power_w` | 정격 % | 80 | 100 | KOSHA Guide E-92-2011 (80% 이내 권장) + KEC 212 (정격 초과 = 과전류) |
| `current` | 정격 % | 80 | 100 | KS C IEC 60947-2 동작 곡선 + 차단기 정격 ≤ 도체 허용전류 |

**산정 논리**:
- `warning_max 80%` — 정격 근처 운전이 지속되면 절연 열화·접속부 발열로 화재 위험. KOSHA Guide 의 "80% 이내 권장" 과 KEC 212 의 "도체 허용전류 = 부하전류 × 1.25" (1/1.25 = 0.8) 의 양면 표현.
- `danger_max 100%` — 정격 초과 = IEC 60947 의 1.13·In 동작 직전 구간. 차단기 동작 전에 운영자 개입 가능한 최후 시점.
- 1.45·In(차단기 강제 동작 영역)을 별도 단계로 두지 않는 이유: 그 시점에는 차단기가 동작해버려 본 모니터링 시스템의 의미가 줄어듦.
- **100% 도달 ≠ 즉각 차단**. IEC 60947-2 의 1.13·In 비동작 구간 때문에 차단기는 한참 더 버틴다. 실질 위험은 *지속 시간에 따른 누적 열 손상*이므로 §3.5 의 3~5초 지속 조건과 결합되어야 운영 가능. 결합 없이 100% 순시값에 즉각 알람을 발하면 부하 스파이크마다 알람이 터져 운영자가 알람 무시 패턴을 학습하게 된다 (false positive 누적 = PoC 실패의 가장 흔한 원인).

### 4.2 저전압/과전압 영역 (양방향)

| measurement_item | 단위 | warning_min | warning_max | danger_min | danger_max | 근거 |
|---|---|---|---|---|---|---|
| `voltage` | 정격 % | 95 | 105 | 90 | 110 | KEC 232.3.9 (저압 옥내 ±5%), IEC 60038 (저압 시스템 ±10% 허용) |

**산정 논리**:
- `warning ±5%` — IEC 60038 의 권장 운전 범위와 일치. KEC 232.3.9 의 5% 와도 정합. 이 범위 내는 정상 운전.
- `danger ±10%` — IEC 60038 저압 시스템 허용 한계. 이를 벗어나면 모터 토크 부족·과전압 절연파괴 위험.
- `min` 위반 = 저전압 (계통 부하 급증, 결상, 케이블 임피던스 이상)
- `max` 위반 = 과전압 (경부하 시 전압 상승, 탭 조정 실패, 무효전력 과다)
- **비대칭 위험에 주의**: 3상 모터의 토크는 V² 에 비례하므로 저전압 10% → 토크 19% 감소 → 정지/슬립 증가 → 과전류 → 권선 과열로 이어지는 연쇄가 매우 빠르다. 반면 과전압 10% 는 절연 수명을 갉아먹지만 *즉발* 위험은 상대적으로 낮다. 본 표는 표준 ±10% 를 적용했지만, **모터 부하 비중이 높은 채널은 `danger_min` 을 92% 로 보수화하는 옵션**이 §7 Open Questions 에 있다.

### 4.3 정격값 예시 (참고용, 실제 입력 값 아님)

[`CHANNEL_TO_DEVICE`](../../fastapi-server/power/services/power_service.py) 매핑 기준으로 추정한 정격 예시. **실제 정격은 현장 명판 확인 후 후속 작업에서 입력**한다.

| 채널 예시 | 설비 | 추정 정격 (참고) |
|---|---|---|
| 압연기 | 산업용 모터 | 100 kW, 380V 3상, 195A |
| 송풍기 | 환기 팬 | 15 kW, 380V 3상, 30A |
| 집진기 | 산업용 집진 | 30 kW, 380V 3상, 58A |
| 조명 | LED 라인 | 5 kW, 220V 단상, 23A |

상기 값은 **설계 가이드용 예시**이며, 본 문서 채택 시점에 확정된 데이터가 아니다.

## 6. AI 이상탐지와의 협력 시나리오

[`apps/ml/`](../../drf-server/apps/ml/) 의 sklearn Isolation Forest 가 연속 anomaly score 를 산출할 때, 본 임계치와 다음과 같이 결합한다.

| 임계치 결과 | IF 결과 | 해석 / 조치 |
|---|---|---|
| NORMAL | normal | 정상 |
| NORMAL | anomaly | **IF 단독 알람** — 시간대 대비 비정상 패턴, 임계치 이전의 전조 |
| WARNING | normal | 임계치 알람 (예상된 부하 증가일 수 있음) |
| WARNING | anomaly | **알람 우선순위 상향** — 예상 외 부하 증가 |
| DANGER | * | 즉각 알람 (IF 결과 무관) |

학습 데이터 라벨링: 본 임계치 위반 구간을 이상 라벨로 활용 → IF 평가 시 정합성 검증에 사용 가능. 단, 학습 자체는 비지도라 정상 데이터만 사용한다.

**3상 불평형은 IF 의 약점 영역**이다. 채널 간 RMS 비교라는 명시적 도메인 계산이 필요해 비지도 outlier 탐지로는 잡기 어렵다. 따라서 별도 규칙 — 예: 동일 설비의 3채널 RMS 평균에서 임의 한 채널이 ±10% 이상 이탈 시 warning — 으로 처리하고, IF 는 시간대 패턴 보조 역할만 한다.

## 7. 결정 보류 사항 (Open Questions)

> 순시 vs 평활(이전 #3) 과 회복 시 히스테리시스(이전 #5) 는 §3.5 의 권고 기본값으로 흡수되었다.

1. **정격값 저장 위치** — 옵션 A(PowerDevice) vs 옵션 B(채널별 ThresholdGroup). 후속 설계 미팅 필요.
2. **야간/주말 별도 임계치** — 휴무 시간대에는 정격 30% 만 넘어도 이상이지만 본 문서는 단일 시간대 기준만 정의. AI 이상탐지가 시간대 패턴을 커버할 수 있는지에 따라 결정.
3. **3상 불평형 처리** — 별도 규칙(채널 간 RMS 비교)으로 처리하고 IF 는 보조 역할 (§6 참조). 규칙의 임계 비율(예시로 든 ±10%)과 적용 대상 채널 묶음 정의는 후속 작업.
4. **저전압 모터 회로의 `danger_min` 보수화** — 표준은 90% 이나 모터 부하 비중이 높은 채널은 92% 로 별도 운영하는 옵션. V² 비례에 의한 토크 19% 감소 → 권선 과열 연쇄를 고려하면 보수적 적용이 안전. 현장 모터 부하 분포에 따라 결정.

## 2. 요구사항 정의서

### [REQ-CM-07] 가스 센서 데이터 수신

**분류** 기능적 요구사항
**중요도** 상

**기능 목적**
에어위드 가스 센서로부터 전송되는 측정값을 FastAPI가 수신하여 검증하고, 서버 기준으로 위험도를 재계산한 뒤 DRF로 전달한다.

**요구사항 상세 설명**

- 센서(또는 dummy_sender.py)가 1초 주기로 `POST /api/sensors/gas`를 호출한다.
- 수신된 `status` 값은 무시하고, 서버가 가스값 기준으로 직접 재계산한다. (센서 오작동·조작 방어)
- `timestamp`에 timezone 정보가 없으면 UTC로 자동 변환한다. (naive datetime 허용하지 않음)
- 가스 9종 개별 위험도(`co_risk`, `h2s_risk` 등)를 서버에서 계산하여 응답 및 DRF 전달에 포함한다.
- `lel`(폭발하한계)은 임계치가 미정의이므로 위험도 계산에서 제외하고, 수집만 한다.
- DRF 연결 실패 시 에러 코드별로 명확한 HTTP 상태코드를 반환한다.

**백엔드 처리 및 인터페이스**

- `schemas/sensors.py` — `GasDataPayload` Pydantic 스키마로 수신값 검증 및 `status` 재계산
- `core/gas_thresholds.py` — 임계치 기준값, `evaluate_single_gas()`, `calculate_individual_risks()`, `calculate_gas_status()`
- `routers/sensors.py` — 엔드포인트 정의, httpx로 DRF `POST /api/monitoring/gas/` 호출
- `core/config.py` — `DRF_BASE_URL` 환경변수 관리

**예외 사항 및 비고**

- 미등록 장치(device_id 없음): DRF가 400 반환 → FastAPI가 404로 전달
- DRF 연결 불가: 503 반환
- DRF 응답 지연(5초 초과): 504 반환
- DRF 기타 오류: 502 반환

**선행 조건 및 의존성**

- `GasSensor` 테이블에 해당 `device_id`가 `is_active=True` 상태로 등록되어 있어야 함
- DRF 서버가 실행 중이어야 함 (`.env`의 `DRF_BASE_URL` 설정 필요)

### [REQ-MN-03] 가스 데이터 DB 저장

**분류** 기능적 요구사항
**중요도** 상

**기능 목적**
FastAPI로부터 전달받은 가스 측정값을 DRF가 검증하고 `GasData` 테이블에 저장한다.

**요구사항 상세 설명**

- `device_id` 문자열을 받아 `GasSensor.objects.get(device_id=..., is_active=True)`로 FK를 해결한다.
- 가스 측정값 9종, 개별 위험도 9종, 원본 페이로드(`raw_payload`)를 저장한다.
- `lel`은 DRF 모델에 컬럼이 없으므로 `raw_payload` JSONField에 원본 페이로드 전체로 보관한다.
- `max_risk_level`은 `GasData.save()` 시 9종 위험도 기준으로 자동 계산되므로 별도 전달하지 않는다.
- 저장 완료 후 `GasSensor.last_reading`을 `measured_at` 값으로 갱신한다.

**백엔드 처리 및 인터페이스**

- `apps/monitoring/serializers/gas_data.py` — `GasDataCreateSerializer` (device_id → FK 해결, 데이터 검증)
- `apps/monitoring/views/gas_data.py` — `GasDataCreateView` (POST 엔드포인트)
- `apps/monitoring/urls.py` — URL 라우팅
- `config/urls.py` — `api/monitoring/` 경로 등록

**예외 사항 및 비고**

- 미등록 장치(`GasSensor.DoesNotExist`): 400 반환 `{"device_id": "등록되지 않은 장치입니다: {device_id}"}`
- 인증 없이 FastAPI 내부 호출만 허용 (`authentication_classes = []`, `permission_classes = []`)

**선행 조건 및 의존성**

- `GasSensor` 테이블에 해당 장치가 사전 등록되어 있어야 함
- `GasData` 테이블 마이그레이션 적용 완료 상태여야 함

## 3. API 명세서

### CM-07 — FastAPI 수신 엔드포인트

#### `POST /api/sensors/gas`

**기능** 가스 센서 측정값 수신 및 위험도 재계산
**사용자** 에어위드 센서 장비 / dummy_sender.py
**메서드** POST
**URL** /api/sensors/gas

**Request**
Content-Type: application/json

```json
{
  "timestamp": "2026-04-22T10:00:00+00:00",
  "device_id": "63200c3afd12",
  "device_name": "63200c3afd12",
  "location": { "x": 140, "y": 160 },
  "o2": 20.15,
  "co": 12,
  "co2": 560,
  "h2s": 4,
  "lel": 2,
  "no2": 1.2,
  "so2": 0.8,
  "o3": 0.02,
  "nh3": 8,
  "voc": 0.3,
  "status": "normal"
}
```

**Response**

Success: 200 OK
```json
{
  "received": true,
  "device_id": "63200c3afd12",
  "status": "normal",
  "co_risk": "normal",
  "h2s_risk": "normal",
  "co2_risk": "normal",
  "o2_risk": "normal",
  "no2_risk": "normal",
  "so2_risk": "normal",
  "o3_risk": "normal",
  "nh3_risk": "normal",
  "voc_risk": "normal"
}
```

Error: 404 Not Found
```json
{ "detail": "등록되지 않은 장치입니다." }
```

Error: 503 Service Unavailable
```json
{ "detail": "DRF 서버에 연결할 수 없습니다." }
```

Error: 504 Gateway Timeout
```json
{ "detail": "DRF 서버 응답 시간 초과." }
```

**설명**

- 수신된 `status`는 무시하고 서버에서 가스값 기준으로 재계산한다.
- `timestamp`에 timezone 없으면 UTC로 자동 변환한다.
- `lel`은 위험도 계산에서 제외하고 DRF의 `raw_payload`에 저장된다.

### MN-03 — DRF 저장 엔드포인트

#### `POST /api/monitoring/gas/`

**기능** 가스 측정값 DB 저장
**사용자** FastAPI (내부 서버 간 호출)
**메서드** POST
**URL** /api/monitoring/gas/

**Request**
Content-Type: application/json

```json
{
  "device_id": "63200c3afd12",
  "measured_at": "2026-04-22T10:00:00+00:00",
  "co": 12, "h2s": 4, "co2": 560,
  "o2": 20.15, "no2": 1.2, "so2": 0.8,
  "o3": 0.02, "nh3": 8, "voc": 0.3,
  "co_risk": "normal", "h2s_risk": "normal", "co2_risk": "normal",
  "o2_risk": "normal", "no2_risk": "normal", "so2_risk": "normal",
  "o3_risk": "normal", "nh3_risk": "normal", "voc_risk": "normal",
  "raw_payload": { "timestamp": "...", "lel": 2, "...": "..." }
}
```

**Response**

Success: 201 Created
```json
{ "id": 1, "received": true }
```

Error: 400 Bad Request
```json
{ "device_id": "등록되지 않은 장치입니다: 63200c3afd12" }
```

**설명**

- `device_id`로 `GasSensor` FK를 조회한다. 미등록 또는 `is_active=False`이면 400 반환.
- `max_risk_level`은 `GasData.save()` 시 9종 위험도 기준으로 자동 계산된다.
- 저장 완료 후 `GasSensor.last_reading`을 `measured_at`으로 갱신한다.

## 4. 흐름도

```
[dummy_sender.py / 에어위드 센서]
        │  POST /api/sensors/gas (1초 주기)
        ▼
[FastAPI — routers/sensors.py]
  1. GasDataPayload 검증 (schemas/sensors.py)
     - timestamp timezone 보정
     - 가스값 범위 체크
  2. status 재계산 (core/gas_thresholds.py)
     - calculate_gas_status() → 전체 위험도
     - calculate_individual_risks() → 9종 개별 위험도
  3. DRF 포워딩 페이로드 구성
     - lel 제외, measured_at으로 필드명 변환
     - raw_payload에 원본 전체 포함
        │  POST /api/monitoring/gas/ (httpx, timeout 5s)
        ▼
[DRF — apps/monitoring]
  4. GasDataCreateSerializer 검증
     - device_id → GasSensor FK 조회
     - is_active=True 확인
  5. GasData.objects.create()
     - save() 시 max_risk_level 자동 계산
  6. GasSensor.last_reading 갱신
        │
        ▼
[DB — gas_data 테이블 저장 완료]
        │
        ▼
[FastAPI → 센서에게 200 응답 반환]
  {received, status, co_risk, h2s_risk, ...}
```

## 1. 개요

현장 설비(Equipment)를 등록·수정·비활성화할 수 있는 관리자 페이지 기능입니다.
설비는 **공장(Facility) ↔ 설비(Equipment) ↔ 전력장치(PowerDevice)** 계층 구조로 연결되며, 설비 비활성화 시 연결된 전력장치도 함께 비활성화되어 지도 편집 및 메인 대시보드에서 자동 제거됩니다.

---

## 2. 데이터 모델

### 2.1 Equipment 모델 (`apps/facilities/models/equipment.py`)

| 필드 | 타입 | 설명 |
|------|------|------|
| `id` | AutoField (PK) | 내부 식별자 |
| `facility` | ForeignKey → Facility | 소속 공장 (PROTECT) |
| `power_device` | OneToOneField → PowerDevice | 연결 전력장치 (SET_NULL, nullable) |
| `name` | CharField(200) | 설비명 |
| `notes` | TextField | 비고 |
| `is_active` | BooleanField | 사용 여부 (기본값: True) |
| `deactivated_at` | DateTimeField | 비활성화 일시 |
| `created_at` | DateTimeField | 등록 일시 (auto) |
| `updated_at` | DateTimeField | 수정 일시 (auto) |

**설비코드 규칙**: `EQP-{id:03d}` (자동 생성, 수동 지정 불가)

**DB 테이블명**: `equipment`

**인덱스**:
- `idx_equipment_facility_active` — (facility, is_active) 복합 인덱스

### 2.2 연관 모델 관계

```
Facility (FAC-001)
  └── Equipment (EQP-001)  ←→  PowerDevice (전력장치)
  └── Equipment (EQP-002)  ←→  PowerDevice (전력장치)
```

- Facility : Equipment = 1 : N
- Equipment : PowerDevice = 1 : 1 (OneToOne)
- PowerDevice는 Equipment 없이도 존재 가능 (미연결 상태)

### 2.3 Soft Delete 정책

물리 삭제 없이 `is_active=False`로 비활성화합니다.

```python
def deactivate(self):
    self.is_active = False
    self.deactivated_at = timezone.now()
    self.save(update_fields=["is_active", "deactivated_at", "updated_at"])
    if self.power_device_id:
        self.power_device.deactivate()  # 연결된 전력장치 연쇄 비활성화
```

## 3. API 명세

**Base Path**: `/api/`
**인증**: JWT Bearer Token (`Authorization: Bearer <token>`)

### 3.1 설비 목록 조회 / 등록

| | |
|---|---|
| **GET** | `/api/equipments/` |
| **POST** | `/api/equipments/` |

**GET 쿼리 파라미터**

| 파라미터 | 타입 | 설명 | 예시 |
|----------|------|------|------|
| `q` | string | 설비코드(EQP-XXX) 또는 설비명 검색 | `EQP-001`, `테스트` |
| `facility` | int | 공장 ID 필터 | `1` |
| `is_active` | `true`/`false` | 사용 여부 필터 | `true` |
| `order` | string | 정렬 기준 | `-created_at`, `name` |
| `page` | int | 페이지 번호 (기본: 1) | `1` |
| `page_size` | int | 페이지 크기 (기본: 20, 최대: 100) | `20` |

**GET 응답 예시**
```json
{
  "total": 3,
  "page": 1,
  "page_size": 20,
  "results": [
    {
      "id": 1,
      "equipment_code": "EQP-001",

      "facility_code": "FAC-001",
      "facility_name": "공장명",
      "facility_address": "주소",
      "name": "설비명",
      "notes": "비고",
      "manager_name": "담당자명",
      "device_id": "전력장치ID",

      "power_device_id": 3,
      "is_active": true,
      "created_at": "2026-04-29T00:00:00Z"
    }
  ]
}
```

**POST 요청 바디**
```json
{
  "facility": 1,
  "name": "설비명",
  "power_device": 2,
  "notes": "비고",
  "is_active": true
}
```

### 3.2 설비 상세 조회 / 수정 / 비활성화

| | |
|---|---|
| **GET** | `/api/equipments/{id}/` |
| **PUT** | `/api/equipments/{id}/` |
| **DELETE** | `/api/equipments/{id}/` |

**PUT 동작 특이사항**
- `is_active: false → true` 전환 시, 연결된 전력장치가 비활성이면 자동 재활성화
- 새 전력장치를 연결하면서 활성화하는 경우에도 해당 장치를 자동 재활성화
- DELETE는 물리 삭제가 아닌 `is_active=False` 처리

---

### 3.3 설비 일괄 비활성화

| | |
|---|---|
| **POST** | `/api/equipments/bulk-delete/` |

```json
{ "ids": [1, 2, 3] }
```

응답: `{ "deleted": 3 }`

> 각 Equipment마다 `deactivate()` 호출 → 연결된 PowerDevice도 연쇄 비활성화

### 3.4 드롭다운 옵션 API

| 엔드포인트 | 설명 |
|-----------|------|
| `GET /api/facilities/select/` | 활성 공장 목록 (id, facility_code, name) |
| `GET /api/facilities/devices/select/` | 전력장치 선택 목록 |

**전력장치 선택 목록 동작 규칙**

| 모드 | 쿼리 파라미터 | 반환 대상 |
|------|-------------|----------|
| 등록 모드 | 없음 | 활성(`is_active=True`) + 미연결(`equipment=null`) 장치만 |
| 수정 모드 | `?equipment_id={id}` | 다른 설비에 연결되지 않은 모든 장치 (활성/비활성 무관) + 현재 연결 장치 |

> 수정 모드에서 비활성 장치도 표시되며 `is_active: false` 필드와 함께 "(비활성)" 레이블로 구분됨

## 5. 비활성화 연쇄 효과

```
설비(Equipment) 비활성화
    ↓
전력장치(PowerDevice) 비활성화
    ↓
지도 편집 (map-editor)    → PowerDevice.is_active=True 필터로 자동 제거
메인 대시보드             → PowerDevice.is_active=True 필터로 자동 제거
```

**재활성화 흐름**:
```
설비 수정 → is_active=True + power_device 선택
    ↓
연결된 PowerDevice.is_active=False인 경우 → 자동 재활성화 (status=normal)
    ↓
지도 편집 / 메인 대시보드에 자동 복원
```

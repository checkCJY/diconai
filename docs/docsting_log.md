# 주석 정비 작업 로그 (Docstring / Inline Comment Log)

> 작성일: 2026-05-07
>
> 비즈니스 로직에 WHY 중심 주석을 단계적으로 보강하면서, 페이즈별 변경 내역과 작업 중 발견한 코드 의문점을 모두 이 파일에 기록한다.
>
> 컨벤션은 [dev_convention.md §6](dev_convention.md#6-comments--logging) 참조.

---

## 작업 원칙 (요약)

- **순수 주석 작업**: 코드 로직(변수명·분기·임계치 값)은 어떤 것도 수정하지 않는다.
- **주석은 코드와 정확히 일치**: 추측·짐작 금지. 의도가 불명확하면 주석을 달지 않고 아래 "코드 의문점" 섹션에 기록한다.
- **코드 문제점 발견 시 직접 수정 X**: 본 파일의 "코드 의문점" 섹션에만 적재 → 사용자가 별도 작업으로 처리한다.

# 모델 변경사항 (2026-04-21)

## 개요

`models_fix.md` 진단서 기반으로 4개 모델 파일을 수정했습니다.
`SystemLog`는 모델 변경 없음 — serializer 레벨 validator는 별도 작업 예정.

---

## 1. `monitoring/models/gas_data.py` — 대규모 수정

### GasData

| 항목 | 기존 | 변경 |
|---|---|---|
| 측정값 저장 | `measurements = JSONField` | 가스 9종 개별 `FloatField` (`co`, `h2s`, `co2`, `o2`, `no2`, `so2`, `o3`, `nh3`, `voc`) |
| 위험도 저장 | JSON 내부 `"risk"` 키 | 가스 9종 개별 `CharField` (`co_risk` ~ `voc_risk`) |
| 원본 보관 | `measurements`가 주 저장소 | `raw_payload = JSONField(null=True, blank=True)` 추가 |
| `max_risk_level` 갱신 | 로직 없음 | `_RISK_ORDER`, `_GAS_RISK_FIELDS`, `calculate_max_risk()`, `save()` 자동 갱신 추가 |

**변경 이유**: 시계열 집계(AVG, MAX), 임계치 비교, 대시보드 차트, AI 학습 파이프라인 모두 컬럼 기반 구조에서 인덱스·ORM을 자연스럽게 활용 가능.

### GasDataError

| 항목 | 기존 | 변경 |
|---|---|---|
| `target_gas` 타입 | 자유 문자열 (`help_text`만 있음) | `GasType(TextChoices)` 이너 클래스 추가 후 `choices` 제한 |

**변경 이유**: 자유 문자열은 집계 쿼리 신뢰성이 없음. `GasType` 제한으로 오타/임의값 저장 방지.

---

## 2. `monitoring/models/power_event.py` — 중간 수정

| 항목 | 기존 | 변경 |
|---|---|---|
| `validate_snapshot` 위치 | 모듈 레벨 함수 | 클래스 내부 `@staticmethod`로 이동 |
| 검증 방식 | subset 차집합 비교 | 키별 명시적 범위+타입 검증 (오류 메시지 구체화) |
| `Trigger` 값 | `manual_admin`, `auto_alarm`, `auto_recovery`, `power_outage` | `manual`, `alarm_auto`, `recovery_auto`, `power_cut` (진단서 확정 값으로 통일) |
| `changed_channels` | `default=list` (null 불가) | `null=True` 추가 — `None`(최초 수신) vs `[]`(변경 없음) 구분 |
| `on_channels` / `off_channels` | 없음 | property 추가 — 스냅샷 파싱 로직 중앙화 |
| `trigger` 인덱스 | `idx_pwr_event_trigger_time` 존재 | 제거 — 실제 쿼리 패턴에 없음, write 비용만 증가 |

---

## 3. `monitoring/models/power_data.py` — 소규모 수정

| 항목 | 기존 | 변경 |
|---|---|---|
| `data_type` 복합 인덱스 | 없음 | `["power_device", "channel", "data_type", "-measured_at"]` (`idx_pwr_device_ch_type_time`) 추가 |
| `__str__` | 없음 | `f"ch{self.channel} {self.data_type} @ {self.measured_at}"` 추가 |
| `latency_seconds` property | 없음 | 통신 지연 시간(초) property 추가 |

**인덱스 추가 이유**: "이 장비 ch1의 전력(watt)만 최근 N건" 같은 `data_type` 필터 쿼리가 대시보드에서 자주 발생하는데 기존 인덱스는 채널 전체 스캔이 필요했음.

---

## 4. `facilities/models/devices.py` — 소규모 수정

| 항목 | 기존 | 변경 |
|---|---|---|
| `PowerDevice.clean()` | 없음 | `channel_meta` 구조 검증 추가 |

**검증 내용**:
- 채널 키가 숫자이고 `1 ~ channel_count` 범위인지 확인
- 각 채널에 `"name"` 필드가 있는지 확인

---

## 5. `core/models/system_log.py` — 변경 없음

`old_value` / `new_value` JSONField는 `action_type`마다 구조가 달라 모델 레벨 validator 작성 불가.
→ 향후 `serializers.py`에서 `action_type`별 분기 validator로 처리 예정.

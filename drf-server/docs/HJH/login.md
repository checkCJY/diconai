# 로그인 페이지 (`login.html`)

## 개요

산재 예방 통합 관제 플랫폼의 로그인 화면입니다.
`/accounts/login/` 경로로 접근 시 렌더링됩니다.

---

## 화면 구성 요소

| 번호 | 요소 | 설명 |
|------|------|------|
| ① | 배경 | 파란 그라디언트 (`#1155a6` → `#0d3f82`) 배경 |
| ② | 로고 | 로고 영역 (SVG 아이콘 + 로고 텍스트 박스) |
| ③ | 플랫폼명 | "산재 예방 통합 관제 플랫폼" 텍스트 |
| ④ | 입력란 | 아이디 / 비밀번호 2개 필드 |
| ⑤ | 로그인 버튼 | 유효성 검사 통과 후 API 호출 |
| ⑥ | 문의 문구 | 로그인 관련 문의 안내 텍스트 (`core/constants.py`의 `CONTACT_INFO`로 관리) |

---

## 입력 필드 공통 동작

- **비식별화**: 비밀번호 필드는 `type="password"` 처리
- **지움 버튼**: 각 필드에 문자가 입력되는 즉시 활성화, 클릭 시 해당 필드 초기화 및 포커스 복귀
- **Enter 키**: 모든 필드에서 Enter 입력 시 로그인 버튼 동작 실행 (`form.submit` 이벤트 활용)

---

## 유효성 검사

> 로그인 버튼 클릭 시 모든 필드를 순서대로 검사합니다.
> 검사 실패 시 해당 필드 하단에 에러 메시지를 표시하고 제출을 중단합니다.
> 클라이언트(JS) + 서버(Serializer) 양쪽에서 동일 규칙으로 이중 검사합니다.

### 아이디

| 조건 | 메시지 |
|------|--------|
| 미입력 | 아이디를 입력해주세요. |
| 영문·숫자 이외의 문자 포함 | 아이디는 영문 또는 숫자만 입력할 수 있습니다. |
| 4자 미만 또는 20자 초과 | 아이디를 4~20자로 입력해주세요. |

### 비밀번호

| 조건 | 메시지 |
|------|--------|
| 미입력 | 비밀번호를 입력해주세요. |
| 8자 미만 | 비밀번호를 8자 이상 입력해야 합니다. |
| 영문·숫자·특수문자 중 1가지만 포함 | 비밀번호는 영문, 숫자, 특수문자 중 2가지 이상을 포함해야 합니다. |

---

## 로그인 처리 흐름

1. 클라이언트 유효성 검사 통과
2. `POST /api/auth/login/` 호출 (JSON body: `{ username, password }`)
3. 서버 처리:
   - 계정 잠금 확인 → 잠금 시 `"계정이 잠겼습니다."` 반환
   - 비활성 계정 확인 → `"비활성화된 계정입니다."` 반환
   - 인증 실패 → 실패 횟수 +1 (5회 시 30분 잠금), `"아이디 또는 비밀번호가 올바르지 않습니다."` 반환
   - 인증 성공 → 실패 카운터 초기화, LoginLog 기록, JWT 토큰 반환
4. 응답 성공 → 토큰 localStorage 저장 후 `/dashboard/` 리다이렉트
5. 응답 실패 → 서버 에러 박스에 메시지 표시

---

## 이미 로그인 상태 처리

페이지 진입 시 `localStorage`에 `access_token`이 있으면 `/api/auth/me/`를 호출해 유효성을 검증합니다.
- 유효한 토큰 → `/dashboard/` 이동
- 만료/무효 토큰 → 토큰 전체 삭제 후 로그인 화면 유지
